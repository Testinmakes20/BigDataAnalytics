Citing the Qualitas Corpus

The main work to cite is:

Ewan Tempero, Craig Anslow, Jens Dietrich, Ted Han, Jing Li, Markus Lumpe, Hayden Melton and James Noble 'Qualitas Corpus: A Curated Collection of Java Code for Empirical Studies' 2010 Asia Pacific Software Engineering Conference (APSEC2010), pp336–345, December 2010.
The BibTeX entry is at the bottom of this page. A preprint (what was submitted to APSEC) is available as PDF. The final version appears in the APSEC 2010 Proceedings.

The preferred citation is to use the APSEC paper and identify the version used. If that is not appropriate for some reason, then the citation to use to identify a particular release used in a study is:

Qualitas Research Group, Qualitas Corpus Version <versionid>, qualitascorpus.com. The University of Auckland, <month> <year>.
where <versionid> is the release date (using ISO 8601) given for the particular release used and <month> and <year> are the month and year of the release date. For example, if the version used is distributed as QualitasCorpus-20090202 then cite as:
Qualitas Research Group, Qualitas Corpus Version 20090202, qualitascorpus.com. The University of Auckland, February 2009.
